# ゲーム体験改善計画

## 問題点
1. **全カード選択が最適解** - カード使用にコストがかからないため戦略性がない
2. **「失敗」表示がUXとして悪い** - ダメージを意図的に受け入れても「失敗」と表示される
3. **ダメージエフェクトが不足** - 負けた感じがしない
4. **バランス問題** - クリアが困難すぎる

## 実装フェーズ

### Phase 1: カード使用コストの導入 ✅
**目的**: 戦略的なカード選択を促す

**変更内容**:
- `ChallengeResolutionService.ts`: カード使用時にコスト消費
- しかし現状ルールブックを確認すると「カードを使用（コストとして活力を消費）」とあり、
  コスト0のカードもあるため、ゲームバランスで調整する方針に変更
- **決定**: コスト消費はすでにルールとして存在するが、チャレンジへの参加は任意なので
  「パワー不足でもダメージを受けて次に進む」選択肢を明示する

### Phase 2: 「失敗」→「ダメージ」表現の改善 ✅
**目的**: より正確なフィードバック

**変更内容**:
- `ChallengeResolutionService.ts`: メッセージを改善
  - 成功: 「チャレンジ成功！」
  - ダメージ: 「ダメージを受けた」(パワー不足時)
- `ChallengeResult`: `resultType` を追加 ('success' | 'damage_taken')
- UIコンポーネント: 表示を更新

### Phase 3: ダメージビジュアルエフェクトの追加 ✅
**目的**: 負けた感覚を視覚的に伝える

**変更内容**:
- `GameBoard.vue`: ダメージ時の画面エフェクト
  - 画面シェイク (CSS animation)
  - 赤フラッシュオーバーレイ
  - 活力バーのパルスアニメーション
- 大ダメージ(10以上)時は強調エフェクト

### Phase 4: バランス調整 ✅
**目的**: クリア可能性を上げる

**変更内容**:
- 初期活力を110 → 120に
- 保険料コストを全体的に軽減
- 成功時のボーナスを増加

### Phase 5: テスト & デプロイ ✅
- ビルド確認
- ローカルテスト
- GitHub Pages デプロイ
